# 村田鉄筋㈱ 加工数量分析システム

## プロジェクト概要
- **名称**: 村田鉄筋㈱ 加工数量分析システム
- **目的**: 本社工場・第二工場の日々の加工数量を入力し、日別・月別・年間で加工実績を分析する業務アプリ
- **主な機能**:
  - 加工数量の毎日入力（部位別 10 区分）
  - 工場区分別（本社工場 / 第二工場 / 全体合算）の集計
  - ダッシュボード、日別・月別・年間・工場比較の各分析画面
  - CSV / PDF 出力
  - ログイン（管理者 / 一般ユーザー）と権限制御
  - 月間目標設定と達成率表示
  - 前日コピー機能、重複登録チェック、低数量自動色付け
  - **人員名登録 / 人員別分析** (v1.2): 加工実績ごとに人員名を複数登録し、人ごとの加工量・参加日数・部位別構成を分析
  - **人工（man_days）入力対応** (v1.3 / プレビュー中): 各人員に 0〜2 の人工値（1日=1.0、半日=0.5、四半日=0.25、残業=1.25〜2.0）を入力可能。人員数 = 人工合計（例: 1+1+0.5 → 2.5人工）。総加工量 ÷ 人工合計で「1人工あたり加工数量」を計算し、部位別数量も人工に応じて按分。CSV/PDF/一覧/人員別分析すべて対応。`worker_names` のみの旧データは各人 1.0 人工として後方互換。
  - **材料搬入トレーラー台数** (v1.7 / 本番反映済 2026-06-18): 加工数量入力画面に「材料搬入トレーラー台数」(0以上の数値・0.5刻みで小数可)を入力できる欄を追加。`X 台` 表記。
    - **DB**: `processing_records` に `trailer_count REAL NOT NULL DEFAULT 0` 列追加(migration 0004)。既存データは 0 として扱う(削除なし)。
    - **入力**: 入力画面の「材料搬入」amberカードに `<input type="number" min="0" step="0.5">` を配置。未入力/null/空文字は 0 台扱い。
    - **編集**: 既存レコードの編集にも反映。前回コピー機能もトレーラー台数をコピー(日付は現在日)。
    - **一覧**: 加工実績一覧の右側にトレーラー台数列。
    - **日別分析**: 「指定期間の集計」を 4 カード化(加工総数量 / 総人工 / 1人工あたり / **トレーラー台数**)。テーブルにも列追加。
    - **月別分析**: 画面上部の **年間トレーラー合計サマリカード** + **専用月別トレーラー集計表** + 詳細テーブル内の **トレーラー台数列**(延べ人工の隣、左から5番目で見落とし防止)の3層構成。
    - **年間分析**: 年間トレーラー台数サマリカード + 年表に列 + **月別推移棒グラフ**(新規追加)。
    - **ダッシュボード**: 今日 / 今月 / 今年 のトレーラー台数 3 カード。本社・第二の内訳もサブ表示。
    - **工場フィルタ反映**: 本社工場 / 第二工場 / 全体合算 のすべてで `SUM(COALESCE(trailer_count, 0))` 集計が正しく動作。
    - **CSV / PDF 出力**: 全 4 レポート(一覧 / 日別 / 月別 / 年間)に `トレーラー台数(台)` 列追加。PDFは `Trailer` 列 + 合計サマリ。
    - **エラー対策**: null / undefined / NaN はすべて `safeNum()` で 0 化、SQL側も `COALESCE(trailer_count, 0)` で belt-and-suspenders。
  - **日別分析「指定期間の集計カード」** (v1.6 / 本番反映済 2026-06-15): 日別分析画面の上部に、選択中の期間・工場フィルタに応じた **加工総数量 / 総人工 / 1人工あたり加工数量** の3枚のサマリーカードを表示。
    - 加工総数量 = Σ(各日の総加工数量)・カンマ区切り・kg表示
    - 総人工 = Σ(各日の人工合計)・**小数点第3位まで固定表示** (例: 82.300、113.637、4.025)
    - 1人工あたり加工数量 = 加工総数量 ÷ 総人工・カンマ区切り・小数1桁・`kg/人工`
    - 工場フィルタ切替・期間切替で即時再計算、データなし/総人工=0 の場合は `-` 表示でゼロ除算回避
    - 既存の部位別1人工あたり加工数量テーブル・チャート・CSV/PDF出力には一切影響なし
  - **部位別 1人工あたり加工数量** (v1.5 / プレビュー中・本日改修): ダッシュボード/日別/月別/年間/工場比較/人員別の全分析画面で、各部位（10区分）の **1人工あたり加工数量** を表形式・棒グラフで表示。
    - **計算ルール改修 (v1.5 / 重要)**: v1.4 までは「全部位を同じ総人工で割る」誤った計算だったが、v1.5 で **部位別数量比率で総人工を割り振り、その部位別人工数で割る** 方式に修正。
      - **部位別人工数** = `Σ_日(その日の総人工 × その部位の数量 ÷ その日の総加工数量)` — 日ごとに算出して期間合計
      - **部位別1人工あたり** = `Σ部位数量 ÷ Σ部位別人工数`
      - 例: 総人工3人工/総加工量3,000kg で 柱2,000kg・梁1,000kg → 柱の人工数=2、梁の人工数=1 → **柱・梁ともに 1,000kg/人工** （以前は 柱667、梁333 だった）
      - 月別・年間・期間指定は **各日ごとに部位別人工数を計算してから合計** (バックエンドAPI `/api/analytics/parts-per-manday` が日単位で算出)
      - 1.25人工などの小数人工にも対応
    - **表示列**: 部位 / 加工数量 / **部位別人工数** / 1人工あたり加工数量 (v1.5 で "部位別人工数" 列を追加)
    - **全体集計**: 上記4列テーブル + 部位カラーの棒グラフ。工場比較画面では「全体」「本社」「第二」の 3 セクションを並列表示。
    - **人員ごと** (人員名ごとの部位別 1人工あたり加工数量): 人員別分析画面に **人員名 / 部位 / 加工数量 / 部位別人工数 / 1人工あたり加工数量** の long-format テーブルを表示。
      - **人ごとの部位別人工数** = `Σ_日(その人の人工 × 部位数量 ÷ 総加工数量)`
      - **人ごとの部位別加工量** = `Σ_日(部位数量 ÷ 総人工 × その人の人工)` （v1.3 から不変）
      - **人ごとの部位別1人工あたり** = 人ごとの部位別加工量 ÷ 人ごとの部位別人工数
    - **追加グラフ**: 基礎・梁・柱 の 1人工あたりランキング（Top10）3 連 + 任意部位を選べる **部位プルダウン式比較グラフ**（10部位切り替え可能）— いずれも v1.5 の部位別人工数ベースで算出。
    - **絞り込み**: 期間 / 年 / 月 / 日付範囲 / 工場区分 / 部位 / 人員名 すべての既存フィルタに連動。
    - **CSV / PDF 出力**: v1.5 で「部位別人工数」列を追加。人員別CSV/PDFは long-format（人員×部位の行）。
    - **エラー対策**: 総加工数量が 0 / 総人工が 0 / 部位別数量が 0 / 人員名・人工が未入力 / `workers_json=null` の旧データ / データ 0 件 — いずれもクラッシュせず "-" または "0kg/人工" 表示。

### 部位 10 区分

### 計算ルール（v1.3 人工対応）
- `total_qty` = 全 10 部位の合計（kg）
- **`staff_count` の意味が変更**: 人員が登録されている場合は **人工合計（man_days の総和）** を表す（例: 1.0+1.0+0.5 → 2.5）。手入力時は従来通り人数値。
- `qty_per_person`（= 1人工あたり加工数量）= `total_qty ÷ staff_count`（0 の場合は 0、ゼロ除算なし）
- **人工（man_days）**: 各人員ごとに 0〜1 の値。1日=1.0、半日=0.5、四半日=0.25。空欄/非数値→1.0、負値→0、1超→1 にクランプ。
- **人別加工量** = `(その日の総加工量 ÷ 人工合計) × その人の人工` → 人工に応じた按分
- **人別部位別量** = `(その日の部位別量 ÷ 人工合計) × その人の人工` を積算
- **後方互換**: 旧データ（`workers_json=null` で `worker_names` のみ）は各人 `man_days=1.0` として読み込まれる

## 画面構成
1. **ダッシュボード**: 今日・今月・今年の加工量、工場別、部位別円グラフ、目標達成率
2. **加工数量入力**: 日付/工場/人員数/**人員名（複数追加・削除可、行ごとに追加ボタン）**/部位別10項目/備考、前日コピーボタン（人員名も含めてコピー）
3. **加工実績一覧**: 人員名タグ列付き、絞り込み・編集・削除・CSV/PDF 出力、低数量行は赤、高生産性行は緑で表示
4. **日別分析**: 日付ごとの棒グラフ（工場積み上げ）と表
5. **月別分析**: 月ごとの棒グラフと月間サマリ表
6. **年間分析**: 年別、月別推移（1人あたり）、部位別年間集計
7. **工場比較**: 本社 vs 第二の月別・部位別並列比較、構成比表示
8. **人員別分析** (NEW): 期間/年/月/工場/人員名で絞り込み、KPI（総加工量/登録人員数/1人あたり平均/最多加工量者/最多参加日数者）、ランキング表、5つのグラフ（総加工量・参加日数・1日平均・部位別構成・月別推移）、CSV/PDF 出力
9. **月間目標設定**（管理者のみ）

## ユーザーガイド
### 加工数量の入力
1. ログイン後、画面上部の **「加工数量入力」** を開く
2. 日付・工場区分・人員数を入力
3. 各部位の数量を kg 単位で入力（入力中に総加工数量・1人あたりが自動表示）
4. 「前回コピー」ボタンで同工場の前回データを呼び出せます
5. 「登録する」を押すと保存されます（同じ日付+工場で重複は不可）

### 分析の見方
- 各分析画面の右上 **「kg / t」** トグルで単位切り替え
- 工場フィルタで「本社工場のみ」「第二工場のみ」「全体合算」を選択
- 各画面の **「CSV」「PDF」** ボタンで現在の表示内容を出力

### 権限
- 管理者: すべてのデータの閲覧・登録・編集・削除・出力・目標設定
- 一般ユーザー: 入力と自分のデータの閲覧・編集（削除不可）
